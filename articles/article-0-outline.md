# Article 0 Outline

Working title:

> I Don't Need a Smarter Real Estate Agent Yet. I Need a Better Benchmark.

## Core Point

Before improving a real estate proposal agent, I need to define what "good" means in my own business context.

Custom evals are not new. The interesting part is turning domain judgment into an evaluation system that can catch failures a polished proposal may hide.

## Draft Shape

1. Start with the failure mode.
   - A proposal can sound professional and still optimize for the wrong investor goal.

2. Explain why generic benchmarks are not enough.
   - They compare model ability in general.
   - My business problem needs domain-specific judgment.

3. Introduce the three evaluation layers.
   - Purpose Alignment.
   - Strategy Formulation.
   - Proposal Fit.

4. Explain the first milestone.
   - Create synthetic cases.
   - Run a baseline.
   - Find which layer fails first.

5. Set the tone.
   - This is not a finished system.
   - This is a build log for making the evaluation target concrete.

## Closing Line Candidate

I am not starting by asking the agent to become smarter. I am starting by asking myself what I would actually trust it to get right.
